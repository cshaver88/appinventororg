Why is it important that the blocks above work regardless of how many items are in the list?

Since there were four quiz questions in the given blocks, you could have put "4" instead of "length of list" in the blocks above, and the code would work. But you always want code that is maintainable because code, at least if its good, is often changed and remixed. By using the more generic "length of list" above, the programmer sets it up so that she can change the list without modifying the computational code if the number of quiz questions changes. Also, most apps have dynamic user generated data. So if the code were remixed in an app in which the user (quiz writer) builds the list, it will still work.
